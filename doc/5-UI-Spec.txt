The Adaptive Player UI captures learner signals primarily through interaction rather than interrogation, emitting structured events that update the learner model and inform the next learning moment.

 

1. Purpose of this document
This document defines:

UI states and components

User interaction flows

Event capture and data contracts

Design constraints and non-goals

It is intended to be implementation-ready for web development.

2. Product boundary (important)
Adaptive Player is a UI shell, not an agent.

It renders learning moments, captures signals, and calls backend intelligence.

All pedagogical decisions happen server-side.

Frontend responsibility = sense â†’ render â†’ capture â†’ respond.

3. Design principles (non-negotiable)
One cognitive job per screen

One recommendation, one alternative

No mandatory data entry

Signals captured through interaction, not forms

Explain decisions only when asked

Never label learner states in the UI
